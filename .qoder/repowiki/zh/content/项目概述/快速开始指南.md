## 简介

QuantDinger 是一个自托管的量化交易和算法交易平台，专为AI研究、Python策略开发、回测和实时执行而设计。该平台提供了一套完整的量化交易工作流，包括市场分析、策略开发、回测、快速交易和实时执行等功能。

### 核心特性

- **AI驱动的市场分析**：支持多种LLM提供商，提供快速的AI市场分析
- **Python原生策略开发**：支持DataFrame风格和事件驱动两种策略开发模式
- **多市场覆盖**：支持加密货币、美国股票、外汇等多种市场的交易
- **实时执行**：通过统一的执行层连接多个交易所
- **多用户管理**：基于角色的访问控制和用户管理系统
- **商业功能**：会员计划、积分系统、USDT支付等商业化功能

## 系统要求

### 硬件要求

- **最低配置**：4GB RAM，20GB 硬盘空间
- **推荐配置**：8GB+ RAM，50GB+ 硬盘空间
- **CPU**：现代多核处理器

### 软件要求

- **Docker**：20+ 版本（推荐使用Docker Desktop）
- **Docker Compose**：2.20+ 版本
- **操作系统**：
  - Linux：Ubuntu 20.04+，CentOS 8+，Debian 10+
  - macOS：macOS 10.15+（Intel和Apple Silicon）
  - Windows：Windows 10/11（WSL2推荐）

### 网络要求

- 出站网络访问（用于API调用和数据获取）
- 防火墙开放端口：
  - 8888：前端Web界面
  - 5000：后端API服务
  - 5432：PostgreSQL数据库
  - 6379：Redis缓存

## 本地开发环境

如果需要进行开发或调试，可以跳过Docker直接使用Python运行：

#### 步骤1：设置PostgreSQL数据库

```bash
# 创建数据库和用户
sudo -u postgres psql
CREATE DATABASE quantdinger;
CREATE USER quantdinger WITH ENCRYPTED PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE quantdinger TO quantdinger;
\q

# 初始化数据库架构
psql -U quantdinger -d quantdinger -f migrations/init.sql
```

```bash

python run.py
```

#### DataFrame策略开发

```python
# 示例：双移动平均策略
# @param sma_short int 14 短期移动平均周期
# @param sma_long int 28 长期移动平均周期

sma_short_period = params.get('sma_short', 14)
sma_long_period = params.get('sma_long', 28)

df = df.copy()
sma_short = df["close"].rolling(sma_short_period).mean()
sma_long = df["close"].rolling(sma_long_period).mean()

buy = (sma_short > sma_long) & (sma_short.shift(1) <= sma_long.shift(1))
sell = (sma_short < sma_long) & (sma_short.shift(1) >= sma_long.shift(1))

df["buy"] = buy.fillna(False).astype(bool)
df["sell"] = sell.fillna(False).astype(bool)
```

## 常见问题解答

### 1. Docker相关问题

**问题**：容器启动失败或无法访问

**解决方案**：
```bash
# 检查Docker服务状态
sudo systemctl status docker

# 重新构建镜像
docker-compose build --no-cache

# 查看详细错误日志
docker-compose logs backend
docker-compose logs postgres
docker-compose logs redis
```

**问题**：端口冲突

**解决方案**：
```bash
# 修改端口配置
# 在根目录.env文件中设置
FRONTEND_PORT=3000
BACKEND_PORT=127.0.0.1:5001
DB_PORT=127.0.0.1:5433
REDIS_PORT=127.0.0.1:6380
```

### 2. 数据库连接问题

**问题**：数据库连接失败

**解决方案**：
```bash
# 检查数据库连接字符串
# 在.env文件中确认
DATABASE_URL=postgresql://username:password@postgres:5432/quantdinger

# 验证数据库服务状态
docker-compose ps postgres

# 检查数据库日志
docker-compose logs postgres
```

### 3. 密钥安全问题

**问题**：SECRET_KEY仍使用默认值

**解决方案**：
```bash
# 生成新的安全密钥
./scripts/generate-secret-key.sh

# 或使用PowerShell脚本
.\scripts\generate-secret-key.ps1

# 在.env文件中设置
SECRET_KEY=your_new_secure_key
```

### 4. 代理和网络问题

**问题**：外部API调用失败

**解决方案**：
```bash
# 配置代理
PROXY_URL=http://proxy_host:proxy_port

# 或使用SOCKS代理
PROXY_URL=socks5h://127.0.0.1:10808

# 配置SSL证书
LIVE_TRADING_CA_BUNDLE=/path/to/ca-bundle.pem
```

### 5. 性能问题

**问题**：系统响应缓慢

**解决方案**：
```bash
# 调整数据库连接池
DB_POOL_MIN=10
DB_POOL_MAX=100

# 调整并发设置
MARKET_EXECUTOR_WORKERS=12
PORTFOLIO_EXECUTOR_WORKERS=6

# 调整Gunicorn配置
GUNICORN_WORKERS=2
GUNICORN_THREADS=16
```

## 性能优化建议

### 1. 系统资源配置

#### 内存优化

```bash
# 增加Docker内存限制
# 在docker daemon.json中设置
{
  "default-runtime": "runc",
  "runtimes": {
    "runc": {
      "path": "runc",
      "runtimeArgs": []
    }
  },
  "default-shm-size": "2gb"
}
```

#### CPU优化

```bash
# 为容器设置CPU限制
# 在docker-compose.yml中添加
services:
  backend:
    deploy:
      resources:
        limits:
          cpus: '2.0'
          memory: 4G
        reservations:
          cpus: '1.0'
          memory: 2G
```

#### PostgreSQL配置

```bash
# 在docker-compose.yml中调整参数
postgres:

    - "-c"
    - "max_connections=200"
    - "-c"
    - "shared_buffers=512MB"
    - "-c"
    - "effective_cache_size=2GB"
    - "-c"
    - "work_mem=16MB"
```
